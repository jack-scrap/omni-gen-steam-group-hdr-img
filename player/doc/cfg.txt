Config files assign values to various system settings upon startup.

Files are scanned one assignment per-line.

Separated by the assignment operator with the key preceeding and the value following, the form is `key = value`.

# Keys
| Name                 | Operation                                                         | Type    |
|----------------------|-------------------------------------------------------------------|---------|
| `skip_boot`          | Whether to play boot sequence on startup                          | Boolean |
| `screen_width`       | Width of display viewport                                         | Number  |
| `fps`                | Target frames per second                                          | Number  |
| `speed`              | Global speed of transformations                                   | Number  |
| `begin_index_at_one` | Whether to begin indexing line numbers at `0` or `1`              | Boolean |
| `line_count`         | Maximum number of lines visible within console                    | Number  |
| `line_width`         | Maximum number of characters visible per-line in console          | Number  |
| `tab_width`          | Number of tab characters produced on key press                    | Number  |
| `tab_expand`         | Convert tab characters to space characters, based on `tab_width`  | Boolean |
| `hl_line_no`         | Highlight line numbers                                            | Boolean |
| `hl_active_line`     | Highlight active lines, opposite highlighting set by `hl_line_no` | Boolean |
| `show_file_perm`     | Display file permissions in status bar                            | Boolean |
| `show_diff`          | Display when buffer has been updated in status bar                | Boolean |
| `time_format`        | Format of time in status bar                                      | String  |

# Values
Values take one of 3 forms -- booleans, numbers and strings.

## Booleans
Booleans simply take the value "no" or "yes".

`n` - No
`y` - Yes

## Numbers
Numbers are provided in decimal format.

`3`

## Strings
Strings are enclosed in double quotes.

`"Text"`

## Restrictions
Additional restrictions may exist.

For example, some keys may require that numbers are positive, and negative values cannot be provided.

Should an invalid value be required, the program will fail to boot.
